Trust & protection

Security on ESA

ESA is built for Islamic schools handling sensitive student, family, and payment data. Security is layered across authentication, tenant isolation, payments, and audit logging — not added as an afterthought.

How ESA protects your school

Multi-tenant isolation

Each school is a separate tenant. Database queries, API endpoints, and portal views scope data to the logged-in user’s school so staff from one madrasa cannot see another school’s students, fees, or messages.

Role-based access (RBAC)

Five roles — Super Admin, School Admin, Teacher, Student, and Parent — each see only the pages and actions their role allows. Sensitive screens (analytics, fee configuration, exam finalisation) are gated with server-side decorators.

Teacher sign-off

Exam results, homework approval, and Hifz progress only become official after a verified teacher signs off. Parents and students see finalised exam outcomes only — draft marks stay hidden.

Secure payments

Parents pay through Stripe Checkout — card details never touch ESA servers. Stripe webhooks confirm payment with signature verification. School payouts use Stripe Connect so funds route to each school’s own account.

Authentication & account safety

Feature What it does
Hashed passwords Django stores passwords with PBKDF2 — never in plain text. Built-in validators block common and overly short passwords.
Email verification New accounts with real email addresses must enter a six-digit code before using the portal. Demo seed accounts are exempt for assessor walkthroughs.
Password recovery Secure reset flow at /accounts/password-reset/ sends a time-limited link to the user’s registered email.
Session login (portal) Browser users authenticate with Django sessions and CSRF-protected forms. Production cookies are marked Secure on Heroku.
JWT API access REST clients use short-lived access tokens (30 minutes) with refresh rotation (7 days) via SimpleJWT at /api/auth/token/.
Parent–child linking Parents link to students using school-issued codes — not open search — so families cannot attach to arbitrary pupils.

Application & infrastructure security

  • CSRF protection — every POST form includes a CSRF token; Stripe webhooks are the only exempt endpoint (signature-verified instead).
  • HTTPS on production — Heroku terminates SSL; CSRF_TRUSTED_ORIGINS and secure cookie flags are set when deployed.
  • Secrets in environment variables — SECRET_KEY, Stripe keys, and Gmail App Passwords live in .env locally and Heroku Config Vars in production — never in Git.
  • Default API deny — Django REST Framework requires authentication on API routes unless a view explicitly opens access (e.g. registration).
  • CORS policy — only configured origins may call the API from browsers; defaults restrict to local dev hosts.
  • Audit logging — logins, registrations, timetable changes, and other sensitive actions write to an append-only AuditLog with user, school, IP, and timestamp.
  • WhiteNoise static files — production static assets are served efficiently without exposing source code.

Data flow (simplified)

Most assessor traffic uses server-rendered pages — not a public API exposed to the browser.

  1. 1

    Browser request

    User visits a URL (e.g. /payments/) with a session cookie.

  2. 2

    Middleware checks

    Security, session, CSRF, authentication, and email-verification middleware run before the view.

  3. 3

    Role & tenant gate

    The view confirms role and scopes queries to request.user.school.

  4. 4

    PostgreSQL

    Only rows for that tenant and role are read or written. Payments and sign-offs are idempotent.

What schools should still do

ESA provides strong defaults, but good operational hygiene matters:

  • Rotate demo passwords after assessment — use strong unique passwords for real staff accounts.
  • Keep Stripe webhook secrets and Gmail App Passwords in Config Vars only.
  • Review School Admin access when staff leave — deactivate accounts in Django admin.
  • Run python manage.py test after upgrades; tenant isolation tests are included in the suite.
