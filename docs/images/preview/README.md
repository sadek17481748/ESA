# Responsive design preview screenshots

Save screenshots here for the **Responsive Design Preview** section at the top of `README.md`.

## Naming convention

Use lowercase, hyphenated filenames:

```
01-landing-desktop.png
01-landing-mobile.png
02-login-desktop.png
...
```

Prefix numbers keep screens in README order. Add `-desktop`, `-laptop`, `-tablet`, or `-mobile` when you capture multiple widths for the same page.

## Tips

- Capture from the live site: https://esa-project-2a7a33dfe3fc.herokuapp.com/
- Use Chrome DevTools device toolbar or https://responsivetesttool.com/
- Log in with demo accounts from the README Demo walkthrough before capturing role-specific pages
- PNG or JPG — keep filenames matching the README placeholders so images render on GitHub

When files are ready, tell the project assistant which filename maps to which screen if anything differs from the README list.
